## 三、技術架構（當前）

### 3.1 整體架構圖

```
瀏覽器
  │
  ├─ GET /used-equipment  ──→  Next.js Server Component
  │                               └─ lib/api/firestore.ts ──→ Firebase Firestore
  │
  ├─ POST /api/submit  ──────→  Next.js API Route
  │                               └─ Firebase Admin SDK ──→ Firestore (inquiries / sellerRequests)
  │
  └─ /admin/**  ──────────────→  middleware.ts (auth guard)
                                   └─ 後台 Client Components → /api/admin/* → Firestore
```

### 3.2 Firestore Collections

| Collection | 用途 | 主要欄位 |
|------------|------|----------|
| `machines` | 二手機台庫存 | name, brand, price, costPrice（不公開）, tradingStatus, isActive |
| `inquiries` | 詢價紀錄 | name, phone, email, product, equipmentId, status, adminNote |
| `sellerRequests` | 客戶託售申請 | machineBrand, machineModel, machinePrice, status, adminNote |

### 3.3 後台認證流程

```
POST /api/admin/login
  → 驗證 ADMIN_PASSWORD（env var）
  → 設定 httpOnly cookie: admin_session

middleware.ts
  → 攔截 /admin/**
  → 無效 cookie → redirect /admin/login
```

### 3.4 圖片管理

管理員上傳圖片流程：
1. 上傳至 Google 雲端硬碟（設定「知道連結的所有人可查看」）
2. 複製分享連結貼入後台縮圖欄
3. 系統自動轉換：`drive.google.com/file/d/{ID}` → `lh3.googleusercontent.com/d/{ID}`
